Classes summary
Address Descriptive data for a physical location. May be used as an actual physical address (place to which one could go), or as a container of "address parts" which should be handled as a unit (such as a city-state-ZIP combination within the US).
ClientDetail Descriptive data for the client submitting a transaction.
Localization Identifies the representation of human-readable text.
Notification The descriptive data regarding the result of the submitted transaction.

PostalCodeInquiryReply FedEx Postal Code Inquiry reply.
PostalCodeInquiryRequest FedEx Postal Code Inquiry request.
PostalCodeServiceAreaDescription Postal Code Service Area description.
ServiceAvailabilityOption The descriptive data which identifies an available FedEx Service.
ServiceAvailabilityReply The descriptive data returned for a FedEx service availability request.
ServiceAvailabilityRequest The descriptive data which is used to determine which FedEx Express services are available between an origin and destination. To check the availability of one particular FedEx Express service and packaging type, include the Service and Packaging elements in the request message. Only information regarding that single service and packaging type will be returned from the request. To obtain a list of all available services for a given origin and destination, omit the Service and Packaging elements from the request. In this case the reply will contain every available service.
TransactionDetail The descriptive data that governs data payload language/translations.
VersionId Identifies the version/level of a service operation expected by a caller (in each request) and performed by the callee (in each reply).
WebAuthenticationCredential Two part authentication string used for the sender's identity
WebAuthenticationDetail Used in authentication of the sender's identity.
